# Phase 3 — Primitive Geometry Modeling

## Goal
Provide constructive geometry tools in Rust and a transfer format so models can be sent to the browser for rendering.

---

## Tasks

### 3.1 Mesh Data Structure
- [x] `Mesh` struct: `Vec<[f32;3]>` positions, `Vec<[f32;3]>` normals, `Vec<u32>` indices
- [x] `Mesh::validate()` — checks index bounds, degenerate triangles
- [x] `Mesh::compute_normals()` — flat and smooth variants
- [x] `Mesh::merge(other: &Mesh)` — combine two meshes into one

### 3.2 Primitive Mesh Generators
- [x] `mesh::box_(width, height, depth) -> Mesh`
- [x] `mesh::sphere(radius, segments, rings) -> Mesh`
- [x] `mesh::cylinder(radius, height, segments) -> Mesh`
- [x] `mesh::cone(radius, height, segments) -> Mesh`
- [x] `mesh::plane(width, depth, subdivisions) -> Mesh`

### 3.3 Constructive Operations
- [x] `extrude(profile: &[Vec2], direction: Vec3) -> Mesh`
- [x] `revolve(profile: &[Vec2], axis: Vec3, angle: f32, segments: u32) -> Mesh`
- [x] Boolean union/difference (basic AABB-level; full BSP deferred to later)

### 3.4 Serialization
- [x] Define `MeshData` as a plain struct with `Vec<f32>` and `Vec<u32>` fields
- [x] Derive `serde::Serialize` / `Deserialize` for `MeshData`
- [x] `Mesh::to_mesh_data()` and `Mesh::from_mesh_data()` conversions
- [x] JSON serialization via `serde_json` (for debugging)
- [x] Binary serialization via `bincode` (for performance)

### 3.5 WASM Bridge
- [x] Expose `generate_box(w, h, d) -> String` via `#[wasm_bindgen]` (returns JSON)
- [x] Expose `generate_sphere(r, seg, rings) -> String`
- [x] Expose `generate_cylinder`, `generate_cone`, `generate_plane`
- [x] Expose generic `mesh_to_typed_arrays(mesh_json: &str) -> String` returning flat JSON arrays
- [x] TypeScript type declarations generated by wasm-bindgen

### 3.6 Tests
- [x] Unit tests for each mesh generator (vertex count, index count, no degenerate triangles)
- [x] Round-trip test: `Mesh → MeshData → Mesh` preserves geometry
- [x] WASM bridge unit tests: JSON output, round-trip, error handling

---

## Acceptance Criteria
- All primitive generators produce valid, renderable meshes
- `MeshData` round-trips through JSON and binary without loss
- Browser can receive mesh arrays from WASM and log vertex counts
